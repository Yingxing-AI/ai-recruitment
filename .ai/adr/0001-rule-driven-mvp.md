# ADR 0001: MVP 使用规则驱动，不依赖外部 AI API

日期：2026-06-18

状态：Accepted

背景：

招聘产品 MVP 需要优先验证业务闭环和本地可运行性，不能把外部 AI API 作为上线前提。

决策：

MVP 阶段使用规则驱动和 mock 能力完成候选人摘要、匹配评分和面试建议，暂不把真实外部 AI API 设为默认依赖。

原因：

这样可以优先保证私有化、本地可演示和部署可控，先验证招聘工作流的真实价值。

取舍：

短期内 AI 表现上限低于真实 LLM，但换来更强的可运行性、可测试性和交付确定性。

影响：

后续接入真实 LLM Provider 时，必须保持现有输出结构和业务闭环稳定，不得反向破坏 MVP 可运行性。
