# Knowledge

## 最佳实践

- OSS 准备度至少需要覆盖许可证、README、贡献指南、安全策略、环境变量示例、CI、敏感信息忽略规则和发布前检查清单。
- Docker 构建上下文应排除 `.env`、缓存、上传目录和构建产物，避免本地敏感文件进入镜像上下文。

## 产品认知

- 当前 MVP 的核心价值是先让招聘团队完成“职位 -> 简历导入 -> 候选人 -> AI 初筛/匹配 -> 应聘/面试”的闭环。
- 规则版 AI 能力适合 v0.1/v0.2 验证工作流，后续真实 LLM 接入应保持现有业务输出结构稳定。
- 自然语言工作流在 MVP 阶段先做规则驱动映射更稳妥，能先把用户意图接到既有动作，再逐步替换成真实 LLM 解析。
- 发布流程和安全基线最好同步落到“文档 + 模板 + 运行时校验”三层，单靠说明文档很容易被误用。
- 招聘系统的可追溯性最好直接嵌进业务动作和详情页，单独做日志页面的效果通常不如“列表-详情-日志”一体化。
- 数据留存策略应优先清理中间产物和日志，保留招聘事实数据，避免把回溯能力一起删掉。

## 重要经验

- 文件导入类接口需要显式覆盖空文件、解析失败、引用对象不存在和存储失败等异常路径。
- 示例配置可以保留本地开发默认值，但生产部署文档必须明确要求替换密钥、密码和 CORS 来源。
