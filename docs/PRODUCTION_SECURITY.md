# Production Security Baseline

这是项目在生产或准生产环境下的最低安全要求。

## 配置要求

- `SECRET_KEY` 必须是足够长的随机值，不能使用默认占位符。
- `DATABASE_URL` 不得使用 SQLite。
- `BACKEND_CORS_ORIGINS` 不能包含 `*`。
- 如果使用 MinIO，访问密钥和密钥口令必须替换为生产值。
- 不要在公开日志中记录简历原文、候选人个人信息或 API Key。

## 部署要求

- 生产环境必须通过环境变量提供敏感配置，不要提交到仓库。
- 对上传文件、数据库和对象存储配置备份与访问控制。
- 对外暴露的 Nginx 入口应开启基础安全头。
- 上传接口应设置合理的体积限制，避免无上限接收大文件。

## 运维要求

- 定期轮换密钥。
- 监控失败登录、异常上传和高频接口调用。
- 发布前确认 `CHANGELOG.md`、版本标签和回滚方案已准备好。
