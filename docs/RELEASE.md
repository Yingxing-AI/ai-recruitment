# Release Process

这是项目当前的轻量发布流程，适用于 v0.x 阶段。

## 版本规则

- 采用语义化版本号。
- `main` 分支始终保持可发布状态。
- 每次可交付里程碑先更新 `CHANGELOG.md`，再打版本标签。

## 发布前检查

- 后端测试通过。
- 前端构建通过。
- `.ai/STATUS.md` 已同步当前状态。
- `CHANGELOG.md` 记录了本次变更。
- `README.md` 和相关文档已覆盖新增用户可见行为。

## 发布步骤

1. 确认本次变更是否属于 MVP 范围。
2. 完成代码审核和本地验证。
3. 更新 `CHANGELOG.md` 的 `Unreleased` 内容。
4. 如有必要，更新版本号并创建 git tag。
5. 推送到远端后由 CI 执行后端测试和前端构建。

## 发布后

- 记录本次版本变化到 `.ai/MEMORY.md`。
- 如有重大决策变化，追加 `.ai/DECISIONS.md`。
- 若影响用户行为，更新 README 或页面说明。
