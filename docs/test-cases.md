# Test Cases

## 验收范围

本页用于 Phase 4 V1 产品化验收，重点检查主流程是否稳定。

## 测试前置

- 服务已通过 `docker compose up --build -d` 启动。
- 已执行演示数据种子脚本。
- 前端访问 `http://localhost:3000`，后端访问 `http://localhost:8000`。

## 用例

| 编号 | 场景 | 步骤 | 预期结果 |
| --- | --- | --- | --- |
| TC-01 | 简历上传 | 在“简历导入”页选择候选人、职位并上传 PDF/DOCX/TXT 简历 | 返回上传成功，列表中生成对应简历记录 |
| TC-02 | 简历解析 | 在 AI 招聘页或接口中触发简历解析 | `resumes.parse_status` 更新为 `parsed`，`parsed_json` 有结构化内容 |
| TC-03 | AI 总结 | 对已解析简历触发候选人总结 | 生成摘要、优势、不足和技能信息 |
| TC-04 | 岗位匹配 | 选择职位和候选人生成匹配评分 | 生成分数、匹配等级、推荐结论和风险点 |
| TC-05 | 面试题生成 | 选择职位和候选人生成面试题 | 返回技术、项目、行为三类问题 |
| TC-06 | 候选人状态流转 | 在招聘流程页将候选人从初筛推进到一面、二面、Offer、录用 | 阶段日志写入，流程列表和候选人详情同步更新 |
| TC-07 | 管理仪表盘 | 打开首页查看统计卡片和招聘漏斗 | 职位、候选人、简历、面试中、Offer、录用与漏斗数据正常显示 |
| TC-08 | 实例验证链路 | 运行实例验证测试，串起上传、解析、总结、匹配、面试题和状态流转 | 全链路通过，且新增的日志、应聘和面试记录可查询 |

## 备注

- 当前 AI 全部为规则版结果，不依赖付费外部模型。
- 若验收失败，优先检查数据库、种子数据和后端日志。
